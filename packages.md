# Service Packages

## Package A — Submission Setup Only

Price: **IDR 15,000,000** per application

This package is suitable for developers or application owners who only need initial setup assistance in Google Play Console.

### Package A Scope

```text
1. Create the app record in Google Play Console.
2. Check the package name or `applicationId`.
3. Check the version code and version name.
4. Upload the Android app bundle.
5. Set up internal testing.
6. Create the testing link.
7. Fill in the basic application information.
8. Provide a checklist of remaining requirements that must still be provided by the client.
```

### Package A Limitations

```text
1. Does not include 14-day closed testing.
2. Does not include tester provision.
3. Does not include production access request.
4. Does not include rejection handling.
5. Does not include assistance until production approval.
6. Does not include the creation of a privacy policy, account deletion page, or legal documents.
```

## Package B — Managed Production Submission

Price: **IDR 35,000,000** per application

This package is suitable for clients who want support from the setup stage until the application is successfully submitted and reaches the production stage.

### Payment Scheme

```text
IDR 15,000,000 paid upfront.
IDR 10,000,000 paid after closed testing begins.
IDR 10,000,000 paid after production approval.
```

### Package B Scope

```text
1. Setup the app in Google Play Console.
2. Check the package name or applicationId.
3. Upload the Android app bundle.
4. Set up internal testing.
5. Complete forms based on data provided by the client.
6. Complete the app category and declarations.
7. Input the privacy policy link.
8. Input the account deletion link.
9. Input the store contact details.
10. Set up closed testing.
11. Provide and coordinate 12–20 testers.
12. Submit the production access request.
13. Read and analyze rejections from Google Play.
14. Prepare solution recommendations for the developer/client.
15. Re-upload the bundle after it has been fixed by the developer.
16. Follow up until the application successfully enters Production.
```

### Package B Limitations

```text
1. Prof. NOTA Inc. does not create or fix the application source code.
2. Prof. NOTA Inc. does not create legal documents, privacy policies, or account deletion pages.
3. Prof. NOTA Inc. is not responsible for transactions, refunds, customer support, or user data.
4. Prof. NOTA Inc. does not guarantee approval if the application violates Google Play policies.
5. All technical fixes are the responsibility of the developer/client.
6. All legal and business information must be provided by the client.
7. If the application poses a risk to the publisher account, the process may be delayed, stopped, or rejected.
```

| Stage |                             Work Item |            Fee |
| ----- | ------------------------------------: | -------------: |
| 1     |      Application setup and onboarding |  IDR 7,500,000 |
| 2     |                Internal testing setup |  IDR 5,000,000 |
| 3     |           App content and store setup |  IDR 5,000,000 |
| 4     |              Closed testing operation |  IDR 7,500,000 |
| 5     |             Production access request |  IDR 5,000,000 |
| 6     | Success fee after production approval | IDR 10,000,000 |

### Package C Scope

```text
1. All work included in Package B.
2. Payments are more structured by stage.
3. Suitable for agencies, studios, startups, or owners of multiple applications.
4. Each payment is connected to a clear deliverable.
```

### Package C Limitations

```text
If a stage is delayed because the required data, documents, or technical revisions have not yet been provided by the client, the timeline will also be delayed and will not be considered a delay caused by Prof. NOTA Inc.
```

## Package D — Official Publishing Operator

Price: **IDR 50,000,000 – IDR 75,000,000** per application

This package is suitable for clients who need Prof. NOTA Inc. to act as the official publishing operator for third-party applications.

### Package D Scope

```text
1. All work included in Package B and Package C.
2. Review application risk before acceptance.
3. Manage the application status in the publisher account.
4. Monitor Play Console until the application is stable.
5. Coordinate more intensively with the developer/client.
6. Handle several rejection cycles within a reasonable scope.
7. Document the submission and approval process.
8. Provide further compliance recommendations if there is a policy notice from Google Play.
```

### Package D Limitations

```text
1. A written agreement is required.
2. A publisher account protection clause is required.
3. Prof. NOTA Inc. may reject high-risk applications.
4. Applications that violate the law, deceive users, misuse data, or risk causing account suspension will not be accepted.
5. Application revenue does not go to Prof. NOTA Inc. unless there is a separate agreement.
```
